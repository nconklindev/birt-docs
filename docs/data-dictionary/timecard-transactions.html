Timecard Transactions

The Timecard Transactions entity provides detailed data related to each timekeeping transaction. It provides a row for each employee and each day, for all types of timecard transactions, such as Worked Shift Segment, Paycode Edit, Absence Exception, Historical Correction, and Hours Worked, and Overtime Approvals.

The Timecard Transactions entity enables a manager to view the actual duration and wages related to a timecard transaction. In addition, it makes available the paycode name and associated comments that are related to the transaction. You can also view the data in the Timecard Transactions entity by worked job or worked labor category when it is used with the Worked Job and Labor Category entity, respectively. It also allows you to see the in and out punches for a shift along with geolocation details related to each punch.

Additional Information

The Timecard Transactions entity provides two special Transaction Types named Raw Total and Raw Total by Calendar Day

The Raw Total transaction type returns the raw data generated by the Totalizer at the worked span level. The data is provided by Employee, Paycode, “Worked” Job, “Worked” Labor Category, and “Worked” Work Rule. The Raw Total by Calendar Day transaction type is similar, except that it splits totals for worked spans that cross midnight into separate totals and ignores the employee's assigned Pay Rule
